## Prompt 的三要素模型

一个高质量的 Prompt 包含三个要素：

```
┌─────────────────────────────────────────┐
│  上下文（Context）：你在做什么？         │
│  - 项目背景、当前状态、已有尝试          │
│                                          │
│  意图（Intent）：你想要什么？            │
│  - 明确的目标、期望的输出格式            │
│                                          │
│  约束（Constraints）：有什么限制？       │
│  - 技术栈、风格要求、不能做什么          │
└─────────────────────────────────────────┘
```

缺少任何一个要素都会导致输出质量下降。

## 结构化 Prompt 模板

### 模板一：功能实现

```
我正在 [项目类型]，使用 [技术栈]。

我需要实现 [功能描述]。

要求：
- [具体要求 1]
- [具体要求 2]

参考现有代码：[文件名] 中的 [类/函数名]

不要：
- [不希望的实现方式]
- [要避免引入的依赖]
```

### 模板二：代码修复

```
以下代码有问题：[描述问题]

错误信息：
[粘贴错误信息]

相关代码上下文：[文件名:行号]

期望行为：[应该发生什么]
实际行为：[实际发生了什么]
```

### 模板三：代码审查

```
请审查 [文件名]，重点关注：
- [具体关注点 1，如"安全性"]
- [具体关注点 2，如"性能瓶颈"]

背景：这个文件处理 [功能描述]，
每天处理约 [使用量]，用户包括 [用户类型]。
```

## 上下文注入技巧：@file 与文件引用

### 精准指定文件

与其说"读取相关文件"，不如直接指定：

```
❌ 模糊
> 看看相关的认证代码，帮我添加 OAuth 登录

✅ 精准
> 读取这三个文件后给我建议：
  SecurityConfig.java（安全配置）
  AuthController.java（认证控制器）
  UserService.java（用户服务）
  然后设计 OAuth 登录的集成方案
```

### 分批注入，避免上下文爆炸

```
❌ 一次性
> 读取所有 30 个 Service 文件，然后...

✅ 分批
> 先读取 UserService 和 OrderService，
  分析两者的公共模式

> 好，基于这个模式，我们再看 PaymentService...
```

## 迭代式对话策略

复杂任务用渐进式拆解，而不是一次性长 Prompt：

### 框架优先，细节后补

```
轮次 1：> 帮我设计一个带权限控制的 API 限流系统的架构方案
         （只需要方案，不写代码）

轮次 2：> 方案不错。现在实现第一层：IP 级别的限流
         （Redis + Lua 脚本方案）

轮次 3：> 单元测试？Mock Redis 还是用 Embedded Redis？
```

### 确认再行动

对于不可逆操作，明确要求 Claude 先确认：

```
> 我需要重构 UserService，把所有数据库操作提取到 Repository 层。
  先告诉我影响范围（会修改哪些文件），不要直接开始修改，
  等我确认后再执行
```

## 常见反模式与修正

| 反模式 | 问题 | 修正 |
|--------|------|------|
| `帮我写代码` | 完全没有上下文 | 说明功能、技术栈、约束 |
| `这不对，重写` | Claude 不知道哪里不对 | 说清楚具体哪里有问题 |
| `你之前说的代码有 bug` | 上下文丢失时 Claude 不记得 | 重新提供代码内容 |
| 一次性要求实现整个系统 | 结果难以控制 | 分解为小步骤 |
| `用最好的方式实现` | "最好"是主观的 | 说明你的评判标准（性能/可读性/可维护性） |

## 实战：同一需求的两种 Prompt 对比

**需求**：实现用户登录功能

**差的 Prompt**：
```
帮我写一个登录功能
```

**好的 Prompt**：
```
我在开发一个 Spring Boot 3.3 + MySQL 的后台管理系统。

需要实现管理员登录功能：
- 用户名 + 密码登录（密码用 BCrypt 加密存储）
- 登录成功返回 JWT token（有效期 24 小时）
- 连续 5 次失败锁定账号 15 分钟
- 记录登录日志（IP、时间、是否成功）

参考：
- 已有 AdminUser 实体（读取 AdminUser.java 了解字段）
- 已有 SecurityConfig.java 配置了 Spring Security
- 项目用 Result<T> 统一响应（读取 Result.java 了解格式）

请不要引入新的外部库（已有 jjwt 可用），
先给我一个实现方案再写代码。
```

title: Prompt 改写挑战
difficulty: 中级
---
练习将"糟糕的 Prompt"改写为高质量的结构化 Prompt，并对比输出结果：

**原始 Prompt（糟糕版）**：
1. `帮我优化这个查询`
2. `这个功能有 bug，帮我修`
3. `写一个缓存`
4. `重构这段代码`

**任务**：选其中 2 个，改写为包含上下文、意图、约束的结构化 Prompt，在 Claude Code 中分别执行糟糕版和改写版，对比输出质量。

---hints---
- 改写时，为每个问题虚构一个具体的项目背景（或使用你自己的项目）
- 对比的维度：输出是否可直接使用？是否符合项目风格？需要几轮补充？
- 「帮我优化这个查询」改写示例：「这是一个用户行为分析的 MySQL 查询（读取以下 SQL），目前查询 100 万行数据需要 3 秒，瓶颈在 full table scan。要求：不改变查询结果，给出添加索引的方案和优化后的查询，预期执行时间 < 100ms」

---solution---
改写示例（`帮我修 bug`）：

```
❌ 糟糕版：这个功能有 bug，帮我修

✅ 改写版：
读取 PostService.java 的 getPostsByCategory 方法，
这个方法有以下问题：

错误信息：
LazyInitializationException: could not initialize proxy - no Session

复现步骤：
1. 调用 GET /api/posts?category=tech
2. 分类下有 50+ 篇文章时稳定复现
3. 分类下文章数少时偶发

项目配置：spring.jpa.open-in-view=false

我怀疑是懒加载问题，但不确定具体哪里触发，
请分析原因并提供修复方案（不要改变查询结果集）
```
